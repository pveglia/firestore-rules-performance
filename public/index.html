Firebase Security rules performance

  • Firestore DB contains one document at /conversation/a-conversation.
  • On start, application will fetch that document and load the counter value in memory.
  • At each click, a http triggered function is called with the next counter value.
  • Function will update the document in the db with the provided value.
  • Upon reception of document snapshot, the counter value is appended to the page.
  • After some increases, updates start to arrive ~30 seconds later, while in the DB the value is update much sooner
  • When rules are disabled, updates arrive within a couple of seconds
